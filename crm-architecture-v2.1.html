L1

Presentation Layer

🌐

Web App

React + Module Federation
  • Host shell load plugin UI modules động
  • Lazy loading chỉ plugins tenant đã enable
  • Shared deps (React, UI kit) qua federation
📱

Mobile App

React Native / Flutter
  • Dynamic feature modules per plugin
  • OTA updates cho plugin UI
  • Offline-first + sync queue
⚙️

Admin Console

Tenant + System
  • Tenant: plugins, users, config, billing
  • System: deploy, tenants, monitoring
  • Tách domain + auth hoàn toàn
L2

API Gateway Layer

🔀

Tenant Resolution

  • Resolve: subdomain / custom domain / X-Tenant-ID
  • Cache mapping tại gateway
  • VIP → dedicated instance pool
🔐

Auth + RBAC

  • JWT + refresh token rotation
  • RBAC per tenant, plugin-specific perms
  • API keys cho M2M integrations
  • SSO (SAML/OIDC) cho Enterprise
⏱️

Rate Limiter

  • Sliding window trên Redis
  • Basic: 100, Premium: 500, Enterprise: 2000, VIP: ∞ req/min
  • Burst allowance: 2× trong 5s
🔄

API Versioning

NEW
  • URL-based: /api/v1/, /api/v2/
  • Plugin APIs version theo plugin ver
  • Deprecation headers + contract tests
L3

Business Logic Layer

🧩

Plugin Management

  • Registry: metadata, versions, dependencies
  • Manager: load/unload/upgrade/rollback
  • Sandbox: VM/Worker isolation, resource limits
  • DI Container: scoped per-request injection
  • Event Bus: sync in-process plugin comm
  • Hook Registry: before/after/replace/filter
🔧

Shared Services

  • Email: template engine + queue (async)
  • Storage: S3, tenant-prefixed paths
  • Notification: WebSocket + Push + In-app
  • Search: ES per-tenant index
  • Scheduler: cron, delayed tasks (tenant-aware)
  • MQ Publisher: giao diện publish distributed events
L4

Data Access Layer

Core principle: Plugin KHÔNG THỂ truy cập DB trực tiếp. Mọi query đều qua Query Interceptor — tự động inject tenant_id. Plugin có bug cũng không thể leak data.
🛡️

Query Interceptor

CRITICAL
  • Auto inject WHERE tenant_id = $current
  • Block queries thiếu tenant context
  • Double safety: Interceptor + PostgreSQL RLS
// Plugin viết: ctx.db.query('customers').where({ status: 'active' }) // Interceptor biến thành: SELECT * FROM customers WHERE status='active' AND tenant_id='abc-123'
🗄️

Pool Manager

  • Shared: max 200 conns, standard tenants
  • VIP: dedicated 30 conns per VIP
  • Auto-create VIP pool on upgrade, drain on downgrade
  • Health check 30s, PgBouncer for multiplexing
💾

Cache (3-Level)

  • L1: In-memory, 100MB/inst, LRU, TTL 60s
  • L2: Redis cluster, TTL 5-15min, compressed
  • L3: Database (source of truth)
  • Keys: t:{tenant_id}:...
  • Invalidation: Redis Pub/Sub → clear all L1
📦

Migration Engine

  • Per-plugin migration files
  • Run on plugin enable (transactional)
  • Fail → full rollback, no partial state
  • Down migrations on disable
  • Lock per tenant (prevent concurrent)
L5

Infrastructure

🐘

PostgreSQL

  • Shared DB + RLS (Basic/Premium)
  • Dedicated DB per VIP
  • Metadata DB (tenants, plugins)
  • Read replicas for reporting
⚡

Redis Cluster

  • L2 cache, sessions, rate limits
  • Pub/Sub cache invalidation
  • Distributed locks
  • Sentinel for failover
📨

Message Queue

NEW
  • RabbitMQ (or Kafka at scale)
  • Async: emails, automation, indexing
  • Dead Letter Queue per queue
  • Per-tenant priority
L6

Cross-cutting Concerns

NEW
Vertical layer — xuyên suốt L1-L5. Enforce nhất quán, không phụ thuộc plugin tự implement.
🔒

Security

  • Encryption: AES-256 (rest), TLS 1.3 (transit)
  • PII auto-masking in logs
  • CSRF/XSS/SQLi protection
  • Secret mgmt (Vault/KMS)
📋

Audit Trail

  • Immutable, append-only log
  • Who/What/When + Before/After
  • Tenant cannot delete
  • SOC2/GDPR ready
🚨

Error Framework

  • Hierarchy: System → Tenant → Plugin → Business
  • Plugin error boundary (no cascade crash)
  • Transient vs permanent classification
  • Circuit breaker, retry, bulkhead, fallback
⚙️

Config

  • Global / Tier / Tenant levels
  • Hot reload (no restart)
  • Feature flags, versioned
  • Plugin config validation via schema
L7

Observability

NEW
📝

Logging

  • JSON, always: tenant_id, request_id, plugin
  • Per-tenant log level
  • PII auto-redaction
  • ELK / Grafana Loki
🔍

Tracing

  • OpenTelemetry SDK built-in
  • Req → Gateway → Plugin → DB → Resp
  • Cross-MQ trace propagation
  • Jaeger / Grafana Tempo
📊

Metrics + Alerting

  • Prometheus: latency, throughput, errors
  • Per-tenant + per-plugin dashboards
  • Alerts: P99>500ms, err>1%, DB conn pool

🛡️

Sandbox Engine

CRITICAL

Plugin lỗi không crash process, không access data khác, không vượt resource limits.

class PluginSandbox { execute(core, method, ctx, args) { return runWithLimits({ timeout: ctx.tier === 'vip' ? 10000 : 5000, maxMemory: 50 * 1024 * 1024, maxQueries: 50, allowedAPIs: [ctx.db, ctx.cache, ctx.eventBus, ctx.logger], // BLOCKED: process, fs, net, child_process onError: (err) => { ctx.logger.error('Plugin failed', { plugin: core.name, err }); return { error: 'SERVICE_UNAVAILABLE', fallback: true }; } }, () => core[method](ctx, ...args)); } }

TNT

Tenant Isolation Model

BASIC / PREMIUM — shared infrastructure
┌──────────────────────────────────────────────────────────────┐
│ App Instances: shared pool (any tenant → any instance)       │
│ Database:      shared tables + RLS + Query Interceptor       │
│ Redis:         shared cluster, keys prefixed t:{id}:         │
│ MQ:            shared queues, tenant_id in every message     │
└──────────────────────────────────────────────────────────────┘

ENTERPRISE VIP — hybrid (dedicated + shared fallback)
┌──────────────────────────────────────────────────────────────┐
│ App Instances: dedicated pool (2–3) + shared as fallback     │
│ Database:      dedicated PostgreSQL instance                 │
│ Redis:         dedicated instance, higher memory             │
│ MQ:            priority queues, processed first              │
└──────────────────────────────────────────────────────────────┘

SEC

Security — Defense in Depth

Layer Defense Map:

L1 Presentation   CSP headers, XSS sanitization, CORS whitelist
       │
L2 API Gateway    TLS 1.3, JWT validation, RBAC, Rate limit, IP whitelist (VIP)
       │
L3 Business       Plugin sandbox, input validation, permission checks, CSRF
       │
L4 Data Access    Query Interceptor (auto tenant_id) + parameterized queries + RLS
       │
L5 Infrastructure Encryption at rest (AES-256), network segmentation, mTLS
       │
L6 Cross-cutting  Audit trail, PII masking, secret management (Vault)
       │
L7 Observability  Anomaly detection, security event alerting, access logs
🛡️

Tenant Data Isolation (3 layers)

  • Layer 1 — Query Interceptor: auto-inject tenant_id. Plugin CANNOT bypass.
  • Layer 2 — PostgreSQL RLS: DB-level enforcement. Backup dù interceptor có bug.
  • Layer 3 — Audit + Alert: cross-tenant query attempt → immediate alert.
🔑

Auth Flow

  • Access token: 15min, refresh: 7d, rotation per use
  • Token blacklist in Redis (logout/revocation)
  • M2M: API keys + HMAC signatures
  • Enterprise: SAML 2.0 / OIDC SSO

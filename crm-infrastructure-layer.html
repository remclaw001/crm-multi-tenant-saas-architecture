Redis Cluster — 6 vai trò trong hệ thống

redis roles & topology
┌─ REDIS CLUSTER ── 3 Master + 3 Replica + Sentinel ──────────────────────┐
│                                                                            │
│  ┌──────────────────────────────────────────────────────────────────────┐ │
│  │                                                                      │ │
│  │  Master-1 ←→ Replica-1    (slots 0-5460)                            │ │
│  │  Master-2 ←→ Replica-2    (slots 5461-10922)                         │ │
│  │  Master-3 ←→ Replica-3    (slots 10923-16383)                        │ │
│  │                                                                      │ │
│  │  Sentinel: 3 nodes, auto-failover master → replica                  │ │
│  │  Sizing: 8GB RAM per node (24GB total usable)                       │ │
│  │                                                                      │ │
│  └──────────────────────────────────────────────────────────────────────┘ │
│                                                                            │
│                                                                            │
│  Redis phục vụ 6 VAI TRÒ khác nhau:                                     │
│                                                                            │
│                                                                            │
│  ❶ L2 CACHE — Cache Manager L2 layer                                   │
│  │                                                                         │
│  │  Key pattern:   t:{tenantId}:{plugin}:{entity}:{hash}                  │
│  │  Example:       t:acme-123:care:sales:status=new:page=1               │
│  │  TTL:           5–15 min (per data type)                               │
│  │  Compression:   LZ4 for values > 1KB                                   │
│  │                                                                         │
│  │  Gọi bởi:      L4 Cache Manager (khi L1 miss)                         │
│  │  Traffic:       HIGH (mọi cache miss từ mọi instance)                  │
│  │                                                                         │
│                                                                            │
│  ❷ DOMAIN MAP — Tenant Resolver cache                                  │
│  │                                                                         │
│  │  Key pattern:   domain-map:{subdomain}                                 │
│  │  Example:       domain-map:acme → "acme-123"                          │
│  │  TTL:           5 min                                                   │
│  │                                                                         │
│  │  Gọi bởi:      L2 Tenant Resolver (mỗi request)                       │
│  │  Traffic:       HIGH read, LOW write (chỉ khi miss → query Meta DB)   │
│  │                                                                         │
│                                                                            │
│  ❸ RATE LIMIT — Sliding window counters                                │
│  │                                                                         │
│  │  Key pattern:   rl:{tenantId}  (Sorted Set)                            │
│  │  Members:       timestamps of recent requests                          │
│  │  TTL:           60 seconds (sliding window)                             │
│  │                                                                         │
│  │  Gọi bởi:      L2 Rate Limiter (mỗi request)                          │
│  │  Operations:    ZADD + ZRANGEBYSCORE + ZCARD                           │
│  │  Traffic:       HIGH (3 Redis ops per request)                          │
│  │                                                                         │
│                                                                            │
│  ❹ AUTH TOKEN BLACKLIST — Revoked JWT tracking                         │
│  │                                                                         │
│  │  Key pattern:   blacklist:{jti}                                        │
│  │  Value:         "revoked"                                              │
│  │  TTL:           = token remaining lifetime                              │
│  │                                                                         │
│  │  Gọi bởi:      L2 Auth Guard (mỗi request)                            │
│  │  Operations:    EXISTS (fast, O(1))                                     │
│  │  Traffic:       HIGH read, VERY LOW write (only on token revoke)       │
│  │                                                                         │
│                                                                            │
│  ❺ PUB/SUB CHANNELS — Cross-instance messaging                         │
│  │                                                                         │
│  │  Channels:                                                              │
│  │  ├── cache:invalidate      → Cache Manager L1 sync                     │
│  │  │   payload: { tenant, pattern }                                      │
│  │  │   subscribers: all app instances                                     │
│  │  │                                                                      │
│  │  ├── context:invalidate    → Tenant Context LRU sync                   │
│  │  │   payload: { tenantId } or { all: true }                            │
│  │  │   subscribers: all app instances                                     │
│  │  │                                                                      │
│  │  └── plugin:deployed       → Hot deploy notification                    │
│  │      payload: { pluginName, version }                                  │
│  │      subscribers: all app instances                                     │
│  │                                                                         │
│  │  Pub/Sub KHÔNG persistent — nếu instance down thì miss message.     │
│  │  OK vì: cache/context rebuild tự động ở next request.               │
│  │                                                                         │
│                                                                            │
│  ❻ REALTIME COUNTERS — Analytics plugin fast counters                   │
│     │                                                                      │
│     │  Key pattern:   t:{tenantId}:stats:{metric}                         │
│     │  Example:       t:acme-123:stats:sales_count  → 148                 │
│     │                 t:acme-123:stats:sales_total  → 4250000             │
│     │  Operations:    INCR, INCRBY (atomic, O(1))                          │
│     │                                                                      │
│     │  Gọi bởi:      Analytics sync event handlers                        │
│     │  Nhanh hơn nhiều so với DB aggregate queries.                    │
│     │  Periodic sync: Redis counters → DB snapshots (every 5min).       │
│                                                                            │
└────────────────────────────────────────────────────────────────────────────┘

Dialect & Strategy — DIP Implementations

Dependency Inversion: L4 (Query Builder, Migration Engine) depend on interfaces (ISQLDialect, ITenantStrategy, IMigrationDialect). L5 cung cấp implementations. Swap DB = tạo 3 implementations mới ở L5, zero code change ở L4.
ISQLDialect implementations
ISQLDialect — AST → Parameterized SQL per DB

  L4 TenantScopedDB gọi: dialect.generateSelect(ast) → { text, params }

  ┌──────────────────────┬───────────────────┬──────────────────┬──────────────────┐
  │ Feature              │ PostgresDialect   │ MySQLDialect      │ SQLiteDialect    │
  ├──────────────────────┼───────────────────┼──────────────────┼──────────────────┤
  │ Placeholder          │ $1, $2, $3        │ ?, ?, ?          │ ?, ?, ?          │
  │ Identifier escape    │ "column"          │ `column`         │ "column"         │
  │ RETURNING            │ RETURNING *       │ ✗ (emulated)     │ ✗ (emulated)     │
  │ INSERT get id        │ RETURNING id      │ LAST_INSERT_ID() │ last_insert_rowid│
  │ Upsert               │ ON CONFLICT DO    │ ON DUPLICATE KEY │ ON CONFLICT DO   │
  │ JSON access          │ col->>'key'       │ JSON_EXTRACT()   │ json_extract()   │
  │ ILIKE                │ ILIKE             │ LIKE (default CI)│ LIKE + COLLATE   │
  │ Boolean              │ TRUE / FALSE      │ 1 / 0            │ 1 / 0            │
  │ Full-text search     │ to_tsvector()     │ MATCH AGAINST    │ FTS5             │
  └──────────────────────┴───────────────────┴──────────────────┴──────────────────┘

  Ví dụ cùng AST: { table:'sales', op:'INSERT', data:{title:'Deal',amount:50000} }

  PostgresDialect:  INSERT INTO sales (tenant_id,title,amount) VALUES ($1,$2,$3) RETURNING *
  MySQLDialect:     INSERT INTO sales (tenant_id,title,amount) VALUES (?,?,?)
                    + SELECT * FROM sales WHERE id = LAST_INSERT_ID()
  SQLiteDialect:    INSERT INTO sales (tenant_id,title,amount) VALUES (?,?,?)
                    + SELECT * FROM sales WHERE rowid = last_insert_rowid()
ITenantStrategy implementations
ITenantStrategy — Tenant Isolation per DB

  L4 TenantScopedDB gọi: strategy.injectTenant(ast, tid) → modified AST
                          strategy.configureConnection(conn, tid)

  ┌──────────────────────┬─────────────────────┬───────────────────┬──────────────────┐
  │ Feature              │ PostgresRLSStrategy │ MySQLStrategy     │ SQLiteStrategy   │
  ├──────────────────────┼─────────────────────┼───────────────────┼──────────────────┤
  │ AST injection        │ ✓ WHERE + INSERT    │ ✓ WHERE + INSERT  │ ✓ WHERE + INSERT │
  │ JOIN injection       │ ✓ all joined tables │ ✓ all joined      │ ✓ all joined     │
  │ configureConnection  │ SET app.tenant = tid│ SET @app_tenant   │ N/A (in-process) │
  │ resetConnection      │ RESET app.tenant    │ SET @app_tenant=''│ N/A              │
  │ DB-level isolation   │ ✓ RLS policies      │ ✗ (no RLS)        │ ✗ (no RLS)       │

  if (dbType === 'postgres') {

  const dialect  = container.resolve('ISQLDialect');

  ctx.db = new TenantScopedDB(conn, tenantId, dialect, strategy);

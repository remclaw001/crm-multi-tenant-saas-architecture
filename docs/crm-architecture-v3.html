layer responsibility matrix

L1 PRESENTATION (Front-end)
  • Trách nhiệm: UI/UX, hiển thị dữ liệu, nhận input.
  • Đặc điểm: Client-side (React/Flutter). No business logic.
  • Ví dụ: Web App (:3002), Admin Console (:3000).


L2 API GATEWAY (Entry Point)
  • Trách nhiệm: AuthN, AuthZ, Rate Limit, Routing.
  • Đặc điểm: Sequential middleware pipeline. Fail-fast.
  • Output: Routed to Plugin/Handler.


L3 BUSINESS LOGIC (Core)
  • Vai trò: Trái tim của hệ thống, xử lý nghiệp vụ.
  • Trách nhiệm: Thực thi logic plugin, validate nghiệp vụ,
    tính toán, gửi event.
  • Đặc điểm: Stateless. Chạy trong Sandbox (đối với plugin).
    Tương tác qua Context.
  • Thành phần: Plugin System, Execution Context,
    Shared Services.
  
  
  
  
  
  
  
  
  
  
  
  
  
  
  
  
  
  
    
  
L4 DATA ACCESS (Abstraction)
  • Vai trò: Trung gian bảo vệ và tối ưu hóa truy xuất dữ liệu.
  • Trách nhiệm: Quản lý kết nối (Pooling), Caching (L1/L2),
    Tenant Injection (RLS), Query Interception.
  • Đặc điểm: Developer KHÔNG gọi trực tiếp DB Driver mà gọi
    qua L4 abstraction.
  • Mục tiêu: Che giấu sự phức tạp của hạ tầng multi-tenant.
  
  
  
  
  
  
  
  
  
  
  
  
  
  
  
L5 INFRASTRUCTURE (Stateful)
  • Vai trò: Lưu trữ và duy trì trạng thái hệ thống.
  • Trách nhiệm: Persist data (PostgreSQL), Cache (Redis),
    Queue (RabbitMQ), File (S3).
  • Đặc điểm: Là lớp duy nhất "có trạng thái" (Stateful).
    Cần backup, replication.


L6 CROSS-CUTTING CONCERNS (System-wide)
  • Vai trò: Các chính sách áp dụng toàn cục.
  • Trách nhiệm: Security Policy, Error Handling,
    Config Management, Dependency Injection.
  • Đặc điểm: Áp dụng lên TẤT CẢ các tầng L1-L5.


L7 OBSERVABILITY (Monitoring)
  • Vai trò: Giám sát sức khỏe và hành vi hệ thống.
  • Trách nhiệm: Logging (Logs), Tracing (Spans),
    Metrics (Counter/Gauge).
  • Đặc điểm: Passive, overhead thấp (~1ms).
    Trả lời câu hỏi "Tại sao?".

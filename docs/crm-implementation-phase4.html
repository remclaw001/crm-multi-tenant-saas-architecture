MetricTypeLabelsMô tả

crm_db_pool_connections_waiting Gauge pool_name Requests đang chờ connection (pool exhaustion indicator)
crm_cache_operations_total Counter operation, result Cache get(hit/miss), set(ok), del(ok). Dùng tính hit rate
crm_nodejs_* Various — Node.js default metrics: heap, GC, event loop lag, file descriptors
📌

Label cardinality: Dùng tenant_tier (3 giá trị: standard|vip|enterprise) thay vì tenant_id cho HTTP metrics để tránh high-cardinality explosion. Với hàng nghìn tenants, dùng tenant_id làm label sẽ tạo hàng triệu time series trong Prometheus.

MetricsInterceptor — route normalization
Route normalization logicTypeScript
// Vấn đề: /api/v1/customers/123 và /api/v1/customers/456 phải cùng label // Giải pháp: dùng req.route.path (NestJS route definition với :param) function normalizeRoute(req) { // req.route.path = '/api/v1/:plugin/ping' (defined by @Get(':plugin/ping')) const path = req.route?.path; if (path) return path; // Fallback: truncate URL, strip query string return (req.url ?? '/unknown').split('?')[0]; } // Result: // /api/v1/customers/123 → /api/v1/:plugin/ping (via req.route) // /health → /health // /metrics → /metrics
PoolMetricsCollector lifecycle
Pool gauge update patternTypeScript
@Injectable() export class PoolMetricsCollector implements OnApplicationBootstrap, OnApplicationShutdown { onApplicationBootstrap() { this.collectOnce(); // Immediately this.intervalHandle = setInterval( () => this.collectOnce(), config.POOL_METRICS_INTERVAL_MS // default 15s ); } private collectOnce() { const stats = this.poolRegistry.getStats(); this.setPoolGauges('shared', stats.shared); this.setPoolGauges('metadata', stats.metadata); for (const vip of stats.vipPools) { this.setPoolGauges(`vip-${vip.tenantId.slice(0, 8)}`, vip); } } }

🔗 Grafana datasource linking (Metrics → Traces → Logs)
🔗

Exemplar linking: Prometheus metrics → Jaeger trace. Khi p95 spike, click vào data point trên Grafana → nhảy thẳng sang trace của request đó trong Jaeger.

🔗

Trace → Log correlation: Trong Jaeger UI, click vào span → Grafana tự query Loki với trace_id → hiển thị đúng log lines của request đó.

📋

Promtail label strategy: Chỉ gắn level, tenant_tier, event làm Loki labels (low cardinality). tenant_id và user_id để trong JSON payload — query bằng LogQL filter: | json | tenant_id = "xxx"

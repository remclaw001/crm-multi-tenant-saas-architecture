Phase 9 — Presentation Layer (L1)

Admin Console · Web App · Module Federation · Mobile — UI binds to stable API, không phải ngược lại

Overview
🎯 Phạm vi
3 apps độc lập: frontend/admin/ (Next.js 15 + Shadcn/ui), frontend/web/ (Next.js 15 + TanStack), frontend/mobile/ (Expo 52 + React Native 0.76 + NativeWind). Module Federation liên kết admin ↔ web.
📐 Nguyên tắc thiết kế
UI bind vào API contract (không drive API). X-Tenant-ID header bắt buộc trên mọi request của web/mobile. Admin dùng Authorization: Bearer không cần tenant header (super-admin scope). Token lưu localStorage trên web, expo-secure-store (Keychain/Keystore) trên mobile.

Admin Console (★ Critical)
🏗 Kiến trúc
Next.js 15 App Router — Server Components mặc định, 'use client' chỉ ở components cần hooks.

Route groups: (dashboard)/ bao gồm sidebar layout. Tất cả pages dưới (dashboard)/ kế thừa Sidebar + main layout.

params là Promise<{ id }> trong Next.js 15 — dùng use(params) để unwrap trong Client Components.
📄 Pages
/dashboard/tenants — TanStack Table v8: sort, filter, pagination
/dashboard/tenants/[id] — tenant detail: plan badge, status badge
/dashboard/tenants/[id]/plugins — PluginToggle per plugin (PATCH API)
/dashboard/metrics — MetricsCard grid, refetchInterval: 15s
🔄 TanStack Query v5
getQueryClient() singleton pattern — server renders luôn tạo fresh client, browser reuse singleton. staleTime: 30s. Retry logic: không retry 4xx (ApiError.status < 500), retry 2 lần cho 5xx.

Mutation pattern: useMutation { onSuccess: () => qc.invalidateQueries } để force refetch sau toggle plugin.
📊 TanStack Table v8
Client-side sort + filter + pagination trong TenantTable. useReactTable nhận state: { sorting, globalFilter } từ parent. flexRender cho cả header và cell.

Column defs dùng ColumnDef<Tenant>[] với typed accessorKey — TypeScript enforce column type safety.
🗄 Zustand v5 Stores
useAuthStore — persist middleware vào localStorage. partialize chỉ lưu token + user (không lưu setter functions).

useTenantStore — ephemeral: selectedTenantId, searchQuery, planFilter. Không persist — reset khi refresh. Dùng để sync search input ↔ query key.
🎨 Shadcn/ui pattern
CSS variables trong globals.css: --primary, --muted, --border, etc. dùng HSL values.
cn() = clsx + tailwind-merge — merge Tailwind classes an toàn, overrides predictable.
Components là primitive wrappers quanh HTML elements, không abstract quá nhiều.
Module Federation
🌐 Kiến trúc HOST / REMOTE
Admin Console = HOST — load plugin UIs động qua NextFederationPlugin. URL remotes inject qua env vars (CUSTOMER_PLUGIN_URL, ANALYTICS_PLUGIN_URL).

Web App = REMOTE — expose ContactsList và DealsList để admin hoặc plugin shell có thể lazy-load at runtime mà không cần rebuild host.
📡 Plugin UI Load Pattern
Admin HOST lazy-loads plugin UI: 
const ContactsList = React.lazy(
  () => import('web/ContactsList')
);
// Wraps in Suspense for loading state
<Suspense fallback={<Spinner />}>
  <ContactsList />
</Suspense>
🔑 Shared Singleton
react và react-dom được khai báo singleton: true trong shared config — đảm bảo chỉ có 1 instance React dù load từ remote.

Admin expose ./DesignTokens từ src/lib/design-tokens.ts để plugin UIs import CSS variables mà không bundle lại Tailwind.
⚠️ Next.js 15 App Router + MF
@module-federation/nextjs-mf v8 hỗ trợ App Router với caveats:
— Remote components phải là Client Components ('use client')
— Server Components không thể import từ MF remotes
— remoteEntry.js chỉ được generate phía client bundle (if (!options.isServer) guard trong webpack config)
Web App (CRM cho End Users)
📋 Contacts Page
useContactStore sync search + status filter → queryKey. TanStack Query refetch tự động khi query key thay đổi. ContactsList component dùng TanStack Table v8 với column sort.

Status filter: lead / prospect / customer / churned — map sang status query param cho API.
💼 Deals Page — Board + List view
DealsList hỗ trợ 2 views:
Board view — group theo stage (STAGE_ORDER array), kanban-style columns với deal cards.
List view — flat list với stage badge và deal value.

Pipeline total = data.reduce((s, d) => s + d.value, 0) hiển thị ở header.
✅ Tasks Page
Toggle filter: todo | in_progress | done. useMutation + updateTask({ status: 'done' }) khi click circle icon → optimistically mark done.

formatDue() — humanize due date: "Due today", "2d overdue", "Due in 5d". Priority color: low=slate, medium=amber, high=red.
🌐 API Client — X-Tenant-ID
crmApi nhận AuthCtx { token, tenantId } per-call (stateless, testable). Mọi request tự động inject:
Authorization: Bearer <token>
X-Tenant-ID: <tenantId>

L2 Gateway đọc X-Tenant-ID header → TenantContext.run() → L4 QueryInterceptor scope queries.
Mobile — Expo 52 + React Native 0.76 + NativeWind v4
📱 Expo Router v4 (file-based routing)
Cùng file-based routing pattern như Next.js App Router:
app/_layout.tsx — root QueryClient provider
app/(tabs)/_layout.tsx — bottom tab navigator
app/(tabs)/index.tsx — home dashboard
app/(tabs)/contacts.tsx — contacts list
app/(tabs)/deals.tsx — deals với stage filter chips
🎨 NativeWind v4
className props trên React Native components — Tailwind CSS cho native.
Setup: babel.config.js + metro.config.js đều cần NativeWind preset.

globals.css với @tailwind directives là entry point cho NativeWind preprocessor.
🔐 Secure Token Storage
expo-secure-store — iOS: Keychain, Android: Keystore.
setAuth(token, user) → persist vào secure store
restoreSession() → đọc lại khi app launch
logout() → delete từ secure store

Khác web: không dùng localStorage (không tồn tại trong RN), không dùng AsyncStorage (không mã hoá).
📊 TanStack Query trên Native
Cùng package @tanstack/react-query, cùng useQuery / useMutation API — zero adaptation cần thiết.

FlatList thay vì table/div — virtualized list, scrollable, native performance. ActivityIndicator cho loading state.

Bản đồ L1 — 3 thành phầnL1 Map — 3 Components

Nguyên tắc cốt lõi: L1 là lớp DUY NHẤT user nhìn thấy. L1 hoàn toàn client-side (chạy trên browser). Mọi tương tác đều gọi API tới L2 Gateway — L1 KHÔNG truy cập database, cache, hay MQ trực tiếp. Plugin UI được load ĐỘNG dựa trên tenant đã enable plugins nào.
Core principle: L1 is the ONLY layer users see. L1 is entirely client-side (runs in browser). Every interaction calls the API to L2 Gateway — L1 does NOT directly access database, cache, or MQ. Plugin UI is loaded DYNAMICALLY based on which plugins the tenant has enabled.

Security tại L1 — Client-side DefenseSecurity at L1 — Client-side Defense

L1 là tuyến phòng thủ đầu tiên. Mặc dù backend (L2-L7) có defense in depth, L1 vẫn cần security riêng để ngăn chặn tấn công phía client: XSS, clickjacking, data leak qua browser, token theft.
L1 is the first line of defense. Although the backend (L2-L7) has defense in depth, L1 still needs its own security to prevent client-side attacks: XSS, clickjacking, data leak via browser, token theft.
L1 security
